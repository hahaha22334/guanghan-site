# 项目注意事项

## 一、部署流程总结

### 项目性质

纯静态网站（single-page application），无构建工具/框架，全部内容在 `index.html` 内联 CSS/JS。

### 部署方式

使用 **Cloudflare Pages**，通过 `wrangler` CLI 部署（不要用 Git 集成）。

### 部署步骤

```bash
# 1. 首次部署（已执行）
npx wrangler pages project create guanghan-site --production-branch master

# 2. 每次更新后重新部署
npx wrangler pages deploy . --project-name=guanghan-site
```

### 历史踩坑

| 问题 | 原因 | 解决 |
|------|------|------|
| Git 集成部署失败（`workerd` 119MB 超 25MB 限制） | Cloudflare 新版将 Pages 归入 Workers 体系，构建环境生成的 `workerd` 二进制被当作部署资产 | 改用 `wrangler pages deploy` CLI 部署 |
| 首次 git commit 包含 `node_modules/` | `.gitignore` 未在首次提交前配置 | `git checkout --orphan` 重写历史 |
| 城市概况图片加载失败 | 源图宽仅 1080px，`optimize-images.js` 跳过了 1200w 生成，但 `<picture>` srcset 引用了不存在的 `城市概况-1200w.*` | `srcset` 只保留实际生成的尺寸（400w, 800w） |
| 手机端 Hero 箭头按钮不可点击 | `.hero-content`（`z-index:10`）DOM 顺序在箭头之后，覆盖了按钮；箭头 `z-index` 同为 10，无法穿透 | 箭头 `z-index` 提到 20，高于内容的 10 |
| 触摸滑动不生效 | 事件绑定在 `.hero-slider`，但 `.hero-overlay` 和 `.hero-content` 绝对定位覆盖在 slider 之上，触摸被拦截 | 改为绑定到父元素 `.hero` |

## 二、修改 bug 后如何同步上线

### 标准流程

```bash
# 1. 修改代码（编辑 index.html 等文件）

# 2. 本地预览验证
npx serve .

# 3. 部署上线
npx wrangler pages deploy . --project-name=guanghan-site

# 4. 收到输出即可访问
# https://guanghan-site.pages.dev
```

> 部署需约 1-2 分钟，完成后终端会打印部署 URL。

## 三、代码注意事项

### 3.1 图片管理

```
images/
├── 轮放1-4.jpg        # 源图（原始大图，本地加工用）
├── 城市概况.jpg        # 源图
├── 房湖公园.jpg        # 源图
├── 金雁湖.jpg          # 源图
├── 牛杂火锅.jpg        # 源图
├── optimized/          # 自动化生成（WebP + AVIF 多尺寸）
│   ├── 轮放1-400w.webp
│   ├── 轮放1-800w.webp
│   ├── 轮放1-1200w.webp
│   ├── 轮放1-1920w.webp
│   ├── 轮放1-400w.avif
│   └── ...
└── external/           # 手动下载（原外链图片本地副本）
    ├── 金面具和青铜大面具.jpg
    ├── 青铜面具.jpg
    └── ...
```

**规则：**

- 新增图片源文件放 `images/`，然后运行 `npm run optimize-images` 生成优化版
- HTML 中使用 `<picture>` 引用 `images/optimized/` 下的文件（不要直接引用源图）
- `<picture>` 写法模板：
  ```html
  <picture>
    <source srcset="images/optimized/文件名-400w.avif 400w, images/optimized/文件名-800w.avif 800w"
            sizes="(max-width: 768px) 100vw, 50vw" type="image/avif">
    <source srcset="images/optimized/文件名-400w.webp 400w, images/optimized/文件名-800w.webp 800w"
            sizes="(max-width: 768px) 100vw, 50vw" type="image/webp">
    <img src="images/源图.jpg" alt="描述" loading="lazy">
  </picture>
  ```
- 首页轮播第一张图**不加** `loading="lazy"`（首屏优先加载）
- 外链图片必须下载到 `images/external/`，禁止依赖外部 URL

### 3.2 响应式设计

断点（mobile-first）：

| 断点 | 适配内容 |
|------|----------|
| 默认（手机） | 单列布局 |
| 768px 以上 | 平板优化 |
| 1024px 以上 | 桌面多列布局 |

修改布局时要在三个断点都检查一遍。

### 3.3 外部依赖（CDN）

全在 `index.html` `<head>` 中异步加载：

- **Google Fonts**：Noto Serif SC + Noto Sans SC（`media="print" onload` 异步）
- **Font Awesome 6.4.0**：图标库（同上异步）

**不要添加新的外部 CDN 依赖**。如需图标优先使用 Font Awesome 已有图标。

### 3.4 CSS 规范

- 使用 CSS 变量（`:root` 中定义）：

  | 变量 | 用途 |
  |------|------|
  | `--bronze` / `--bronze-dark` / `--bronze-light` | 主色调（青铜色） |
  | `--gold` / `--gold-light` | 强调色（金色） |
  | `--cream` / `--cream-dark` | 背景色 |
  | `--text-dark` / `--text-light` | 文字色 |
  | `--shadow` / `--shadow-hover` | 阴影效果 |

- 新增颜色值时遵循现有色系，不要引入新色系

### 3.5 JavaScript 要点

- 滚动动画使用 `IntersectionObserver`（`fade-in` / `fade-in-left` / `fade-in-right` 类名）
- Hero 轮播：自动 5s 切换 + 手动按钮 + 触摸滑动
- 移动端导航：汉堡菜单展开/收起
- 新增交互优先用原生 JS，不引入框架

### 3.6 文件大小限制

| 限制项 | 上限 |
|--------|------|
| Cloudflare 单文件 | 25 MiB |
| GitHub 单文件 | 100 MB |
| 总部署文件数 | 20,000（远低于此） |

### 3.7 其他

- `node_modules/` 已加入 `.gitignore`，本地开发依赖 sharp 包
- `package.json` 脚本：
  - `npm run optimize-images` — 重新生成优化图片
  - `npm start` — 本地预览（`npx serve .`）
- 修改后先本地预览再部署
- `_headers` 文件配置了浏览器缓存策略（图片 1 年）
- 不要提交 `images/optimized/` 中未使用的文件
